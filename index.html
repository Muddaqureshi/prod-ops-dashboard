📐 Methodology — 5-Model Ensemble + Inverse-MAPE Weighted Average

Training: 13 months of canonical data (May 2025 – May 2026). Validation: 12-month holdout backtest for main metrics. We use the Darts library for time series forecasting, which provides production-grade implementations of Prophet, LinearRegression, ExponentialSmoothing, and AutoTheta models.

1. Prophet (Facebook/Darts): Additive model with automatic changepoint detection and yearly seasonality. Handles trend shifts and anomalies. Best for Repos (11.8% MAPE) and Actions metrics (50-52% MAPE). Winner on 3 of 6 metrics. Requires 3+ years of data (13 months of clean canonical data).

2. LinearRegression (Darts ML): Time-series regression with trend and seasonal features. Most interpretable model. Excellent on Repos (7-14% MAPE) and PRs (19-22% MAPE). Fails on non-linear patterns (Issues/Commits). Provides transparent baseline for comparison.

3. Monte Carlo Simulation (Custom): 10,000 stochastic simulations using log-normal growth rates sampled from trailing 12 months (μ and σ of month-over-month log-returns). Produces P10/P25/P50/P75/P90 uncertainty bands. Best for PRs (12-17% MAPE) and Commits (26-43% MAPE).

4. ExponentialSmoothing (Darts): Multiplicative triple exponential smoothing with optimized α/β/γ parameters. Fast training (<1s). Best for Commits (21-26% MAPE). Seasonal period = 12 months. Trained on 13 months for main metrics, 13 for Actions).

5. AutoTheta (Darts): Optimized Theta method — decomposes series into trend + seasonal components. Robust to outliers. Best for Issues (23-25% MAPE). Requires 10+ training points; fails on Actions holdout (only 8-9 points after 4-month split).

Per-Metric Model Selection: Each metric uses a tailored subset of the 5 models based on backtest performance. Main metrics use all 5 models.

Weighted Average: Aggregates eligible models using inverse-MAPE weighting from holdout backtest. **Models >60% MAPE excluded per-metric** (Monte Carlo excluded from Actions). Models with lower error get higher weight. This is not an equal average; it rewards accuracy. All forecasts are directional estimates with uncertainty bands, not point predictions. See validation charts below for per-model performance.

💾 Data Sources — Canonical Trino Queries

All data queried from Trino (delta.canonical.* tables) with spam/staff filtering. Clean canonical source of truth for May 2025 - May 2026 (13 months).

📦 Repositories Created canonical · 13 months 
SELECT
  DATE_TRUNC('month', created_at) AS month,
  COUNT(*) AS total_repos
FROM delta.canonical.repositories_all
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND DATE(created_at) = DATE(day)
  AND is_spammy_owner = FALSE
GROUP BY 1
ORDER BY 1;
🔀 Pull Requests Merged canonical · 13 months 
SELECT
  DATE_TRUNC('month', closed_at) AS month,
  COUNT(*) AS total_prs
FROM delta.canonical.pull_requests_all
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND state = 'merged'
  AND DATE(closed_at) = DATE(day)
  AND is_spammy_owner = FALSE
  AND is_staff_owned = FALSE
GROUP BY 1
ORDER BY 1;
📝 Issues Created canonical · 13 months 
SELECT
  DATE_TRUNC('month', i.created_at) AS month,
  COUNT(*) AS total_issues
FROM delta.canonical.issues_all i
JOIN delta.canonical.accounts_all a
  ON i.author_id = a.id
  AND i.day = a.day
WHERE i.day >= '2025-05-01'
  AND i.day < '2026-06-01'
  AND DATE(i.created_at) = DATE(i.day)
  AND a.is_spammy = FALSE
  AND a.is_staff = FALSE
GROUP BY 1
ORDER BY 1;
💾 Commits Created canonical · 13 months 
SELECT
  DATE_TRUNC('month', c.created_at) AS month,
  COUNT(*) AS total_commits
FROM delta.canonical.commits_all c
WHERE c.day >= '2025-05-01'
  AND c.day < '2026-06-01'
  AND DATE(c.created_at) = DATE(c.day)
  AND c.is_spammy_owner = FALSE
  AND c.is_staff_owned = FALSE
GROUP BY 1
ORDER BY 1;
⚡ Actions Workflow Runs canonical · 13 months · proxy via num_unique_workflow_runs 
SELECT
  DATE_TRUNC('month', CAST(usage_month || '-01' AS DATE)) AS month,
  SUM(CAST(num_unique_workflow_runs AS BIGINT)) AS total_workflow_runs
FROM delta.canonical.global_actions_usage
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND CAST(usage_month || '-01' AS DATE) >= DATE '2025-05-01'
  AND CAST(usage_month || '-01' AS DATE) < DATE '2026-06-01'
  AND product_name = 'actions'
  AND is_staff_owned = FALSE
  AND is_spammy = FALSE
GROUP BY 1
ORDER BY 1;
⏱️ Actions Minutes (Actual) canonical · 13 months · using raw_quantity 
SELECT
  DATE_TRUNC('month', usage_month) AS month,
  SUM(CAST(raw_quantity AS BIGINT)) AS actual_minutes
FROM delta.canonical.global_actions_usage
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND usage_month >= DATE '2025-05-01'
  AND usage_month < DATE '2026-06-01'
  AND product_name = 'actions'
  AND unit_of_measure = 'Minutes'
  AND is_staff_owned = FALSE
  AND is_spammy = FALSE
GROUP BY 1
ORDER BY 1;

✅ Why Canonical Tables?

  • No survivorship bias — Complete historical data preserved (deleted/spam repos don't vanish)
  • No bot spam — Proper filtering of github-actions[bot] and spam accounts
  • No pipeline artifacts — Real data only, no backfill spikes
  • No commit duplication — Single-counted via canonical data model
  • Recommended by Data Team — Josh confirmed canonical is source of truth
