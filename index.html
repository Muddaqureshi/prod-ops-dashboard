📊 Data Sources - Canonical Trino Queries

All queries run against delta.canonical.* tables with spam/staff filtering. Date range: May 2025 - May 2026.

Query 1: Repository Creations
SELECT
  DATE_TRUNC('month', created_at) AS month,
  COUNT(*) AS total_repos
FROM delta.canonical.repositories_all
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND DATE(created_at) = DATE(day)
  AND is_spammy_owner = FALSE
GROUP BY 1
ORDER BY 1;
Query 2: Pull Request Merges
SELECT
  DATE_TRUNC('month', closed_at) AS month,
  COUNT(*) AS total_prs
FROM delta.canonical.pull_requests_all
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND state = 'merged'
  AND DATE(closed_at) = DATE(day)
  AND is_spammy_owner = FALSE
  AND is_staff_owned = FALSE
GROUP BY 1
ORDER BY 1;
Query 3: Issues Created
SELECT
  DATE_TRUNC('month', i.created_at) AS month,
  COUNT(*) AS total_issues
FROM delta.canonical.issues_all i
JOIN delta.canonical.accounts_all a
  ON i.author_id = a.id
  AND i.day = a.day
WHERE i.day >= '2025-05-01'
  AND i.day < '2026-06-01'
  AND a.day >= '2025-05-01'
  AND a.day < '2026-06-01'
  AND DATE(i.created_at) = DATE(i.day)
  AND a.is_spammy = FALSE
  AND a.is_staff = FALSE
GROUP BY 1
ORDER BY 1;
Query 4: Actions Workflow Runs (proxy via num_unique_workflow_runs)
SELECT
  DATE_TRUNC('month', CAST(usage_month || '-01' AS DATE)) AS month,
  SUM(CAST(num_unique_workflow_runs AS BIGINT)) AS total_workflow_runs
FROM delta.canonical.global_actions_usage
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND CAST(usage_month || '-01' AS DATE) >= DATE '2025-05-01'
  AND CAST(usage_month || '-01' AS DATE) < DATE '2026-06-01'
  AND product_name = 'actions'
  AND is_staff_owned = FALSE
  AND is_spammy = FALSE
GROUP BY 1
ORDER BY 1;
Query 5: Actions Minutes (actual, using raw_quantity)
SELECT
  DATE_TRUNC('month', usage_month) AS month,
  SUM(CAST(raw_quantity AS BIGINT)) AS actual_minutes
FROM delta.canonical.global_actions_usage
WHERE day >= '2025-05-01'
  AND day < '2026-06-01'
  AND usage_month >= DATE '2025-05-01'
  AND usage_month < DATE '2026-06-01'
  AND product_name = 'actions'
  AND unit_of_measure = 'Minutes'
  AND is_staff_owned = FALSE
  AND is_spammy = FALSE
GROUP BY 1
ORDER BY 1;
Query 6: Commits Created
SELECT
  DATE_TRUNC('month', c.created_at) AS month,
  COUNT(*) AS total_commits
FROM delta.canonical.commits_all c
WHERE c.day >= '2025-05-01'
  AND c.day < '2026-06-01'
  AND DATE(c.created_at) = DATE(c.day)
  AND c.is_spammy_owner = FALSE
  AND c.is_staff_owned = FALSE
GROUP BY 1
ORDER BY 1;

Why Canonical Tables?

  • ✅ No survivorship bias - complete historical data preserved
  • ✅ No bot spam - proper filtering of github-actions[bot] and spam accounts
  • ✅ No pipeline artifacts - real data only, no backfill spikes
  • ✅ No commit duplication - single-counted via canonical data model
  • ✅ Recommended by Data Team - Josh confirmed canonical is source of truth

📐 Methodology — 5-Model Fleet-Validated Ensemble + Inverse-MAPE Weighted Average

Training: 13 months of data (May 2025 – May 2026). Validation: 4-month holdout backtest.

Models trained using Darts time series forecasting library for Prophet, LinearRegression, ExponentialSmoothing, and AutoTheta. Monte Carlo simulation is custom implementation.

Model Ensemble (5 Models):

  1. Prophet (Facebook/Darts): Additive model with automatic changepoint detection and yearly seasonality. Handles trend shifts and anomalies. Best for Repos and Actions metrics. Requires 3+ years of data (we have 13 months for focused recent trend analysis).
  2. LinearRegression (Darts ML): Time-series regression with trend and seasonal features. Most interpretable model. Excellent on Repos and PRs. Fails on non-linear patterns (Issues/Commits). Provides transparent baseline for comparison.
  3. Monte Carlo Simulation (Custom): 10,000 stochastic simulations using log-normal growth rates sampled from trailing 12 months (μ and σ of month-over-month log-returns). Produces P10/P25/P50/P75/P90 uncertainty bands. Best for PRs and Commits.
  4. ExponentialSmoothing (Darts): Multiplicative triple exponential smoothing with optimized α/β/γ parameters. Fast training (<1s). Best for Commits. Seasonal period = 12 months.
  5. AutoTheta (Darts): Optimized Theta method — decomposes series into trend + seasonal components. Robust to outliers. Best for Issues. Requires 10+ training points.

Per-Metric Model Selection:

Each metric uses a tailored subset of the 5 models based on backtest performance. Main metrics use all 5 models where data permits.

Weighted Average:

Aggregates eligible models using inverse-MAPE weighting from holdout backtest. Models >60% MAPE excluded per-metric. Models with lower error get higher weight. This is not an equal average; it rewards accuracy. All forecasts are directional estimates with uncertainty bands, not point predictions.

⚠️ Caveat:

Model accuracy varies by metric and data availability. With 13 months of clean canonical data, forecasts focus on recent growth trends. All forecasts are directional estimates with uncertainty bands, not point predictions.
