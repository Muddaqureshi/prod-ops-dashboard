📐 Methodology — 5-Model Ensemble + Inverse-MAPE Weighted Average

Training: 13 months of canonical data (May 2025 – May 2026). Validation: 12-month holdout backtest for main metrics. We use the Darts library for time series forecasting, which provides production-grade implementations of Prophet, LinearRegression, ExponentialSmoothing, and AutoTheta models.

1. Prophet (Facebook/Darts): Additive model with automatic changepoint detection and yearly seasonality. Handles trend shifts and anomalies. Best for Repos (11.8% MAPE) and Actions metrics (50-52% MAPE). Winner on 3 of 6 metrics. Requires 3+ years of data (41 months available).

2. LinearRegression (Darts ML): Time-series regression with trend and seasonal features. Most interpretable model. Excellent on Repos (7-14% MAPE) and PRs (19-22% MAPE). Fails on non-linear patterns (Issues/Commits). Provides transparent baseline for comparison.

3. Monte Carlo Simulation (Custom): 10,000 stochastic simulations using log-normal growth rates sampled from trailing 12 months (μ and σ of month-over-month log-returns). Produces P10/P25/P50/P75/P90 uncertainty bands. Best for PRs (12-17% MAPE) and Commits (26-43% MAPE).

4. ExponentialSmoothing (Darts): Multiplicative triple exponential smoothing with optimized α/β/γ parameters. Fast training (<1s). Best for Commits (21-26% MAPE). Seasonal period = 12 months. Requires 24+ months (42 available for main metrics, 13 for Actions).

5. AutoTheta (Darts): Optimized Theta method — decomposes series into trend + seasonal components. Robust to outliers. Best for Issues (23-25% MAPE). Requires 10+ training points; fails on Actions holdout (only 8-9 points after 4-month split).

Per-Metric Model Selection: Each metric uses a tailored subset of the 5 models based on backtest performance. Main metrics use all 5 models.

Weighted Average: Aggregates eligible models using inverse-MAPE weighting from holdout backtest. **Models >60% MAPE excluded per-metric** (Monte Carlo excluded from Actions). Models with lower error get higher weight. This is not an equal average; it rewards accuracy. All forecasts are directional estimates with uncertainty bands, not point predictions. 📋 Data Quality Notes

Canonical Tables: Repos, PRs, and Issues use canonical tables (*_current suffix) that filter out spam, deleted entities, and test data. This matches the official GitHub metrics shown in admin dashboards.

Commits: Uses github_collab_commit_contributions_current (latest snapshot only) to avoid double-counting. Previous version used non-current table which counted commits twice due to snapshot overlap.

Actions Metrics: Limited to 13 months of history (Jun 2025–Jun 2026). Actions workflow runs table only became reliable in mid-2025. Earlier data excluded due to instrumentation gaps.

Data Freshness: All queries run through May 31, 2026. Forecasts start June 1, 2026 and extend through December 2026 (8 months ahead).
